## 图表数据重复 bug 修复
- 22:10 发现 `charts.json` 中时间序列数据有严重重复：同一日期出现多次（如 5/22 重复12次，5/28 重复3次）
- **根因**：`metrics_ts` 表按 `(metric_id, as_of, snapshot_file)` 做 unique，每次 `run.py` 生成新 snapshot 都会插入新行。`charts.py` 中多个 SELECT 查询没有去重，导致同一日期返回多条记录
- **修复**：修改 4 个查询函数，全部使用 `SELECT ... WHERE id IN (SELECT MAX(id) ... GROUP BY as_of)` 去重，只取每个日期最新插入的记录
  - `fetch_fred_series_points()` — FRED 数据查询
  - `_nyfed_series_from_db()` — NY Fed 数据查询（2 条 SELECT）
  - `_rrp_from_db()` — RRP 查询
  - `_tga_from_db()` — TGA 查询
- 修复后重新生成 `charts.json` 和 `dashboard_data.json`，验证无重复：
  - `anchor_spreads_7d` SOFR_ANCHOR: 6 points, duplicates: none
  - `anchor_spreads_30d` SOFR_ANCHOR: 22 points, duplicates: none
- 注意：5/29（周五）SOFR 数据仍缺失，导致 SOFR_ANCHOR 跳过该日期——这是原始数据缺失，非去重问题
