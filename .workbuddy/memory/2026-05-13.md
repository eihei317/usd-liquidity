# 2026-05-13 工作记录

## 美元流动性监测前端加载问题修复

### 问题
用户执行"更新数据并且打开"后，前端页面报错：
1. 首先报 404：`../../output/latest/dashboard_data.json` 路径解析失败
2. 修复路径后报 `Failed to fetch`：预览面板内置浏览器禁止 fetch 请求

### 修复方案
彻底绕过 fetch，将数据通过 `<script>` 标签内联：

1. **修改 `scripts/usd_liquidity_brief.py` 的 `write_outputs()` 函数**：
   - 在写入 `latest/` 目录后，额外生成 `frontend/usd-liquidity-monitor/data.js`
   - `data.js` 内容：`window.DASHBOARD_DATA = {...}; window.ANALYSIS_DATA = {...};`

2. **修改 `frontend/usd-liquidity-monitor/index.html`**：
   - 在 `app.js` 之前加载 `data.js`（带缓存戳）

3. **修改 `frontend/usd-liquidity-monitor/app.js` 的 `init()` 函数**：
   - 优先读取 `window.DASHBOARD_DATA` / `window.ANALYSIS_DATA`
   -  fallback 到 fetch（兼容直接打开 HTML 文件的场景）

### 结果
- 每次运行脚本自动生成 `data.js`，无需手动复制文件
- 页面通过 `<script>` 标签加载数据，完全绕过 fetch/CORS 问题
- HTTP 服务从工作区根目录启动，端口 8088

### 技术笔记
- 预览面板的内置浏览器对 `fetch()` 到 localhost 有限制，表现为 `Failed to fetch`
- zsh 中 `?` 是通配符，curl URL 必须加引号
- Python 脚本中 `ROOT = Path(__file__).resolve().parents[1]` 指向工作区根目录
