# MEMORY

## 项目约定

- 2026-05-04：美元流动性简报脚本应采用三段式模型流水线：模型先阅读介绍文档；脚本抓取官方/权威接口数据并保存结构化 snapshot JSON；再把 snapshot JSON 与分析 prompt 组装成 model_input JSON 交给模型分析。最终人读简报要美观、简洁，不应附带原始模型 context 或完整 prompt；利率表固定为“利率 / 含义 / 边际变化”，所有指标名统一为“英文（中文翻译）”；数据频率与环比口径应贴在对应指标后面或作为同一卡片/同一行元信息展示，避免日频、周频、事件驱动和政策阶梯型数据混比；图表默认覆盖最近一周和一月核心指标。下一阶段已确定采用 A 方案：生成“美元流动性监测”前端界面，米白色 Claude 风格；尽量减少分析以外的模型操作，模型只输出结构化 analysis.json，前端用稳定模板确定性渲染；日常运行不让模型改前端，只有产品迭代时再改前端代码。当前已落地 `frontend/usd-liquidity-monitor/`，脚本输出 `output/latest/dashboard_data.json`、`analysis.json`、`charts.json` 供前端读取。
- 2026-05-08：美元流动性监测的利率覆盖从“联储-银行 / 银行间 / 联储-货币基金”短端资金框架，扩展为“短端资金市场 + 债券定价锚 + 信用/证券市场传导”。传导链固定为 Fed负债端水位 -> 政策锚/准备金边际 -> 银行间无抵押融资 -> 回购融资/抵押品链条 -> 债券定价锚/收益率曲线 -> 离岸美元 -> 信用市场/金融条件 -> 证券市场风险偏好。核心新增指标包括 DTB3、DFII10、BAMLC0A0CM、VIXCLS；新增衍生信号包括 TBILL_CASH_YIELD、NOMINAL_10Y、REAL_10Y、IG_CHANGE、VIX_RISK。
- 2026-07-14 / 2026-07-24 更新：美债收益率主框架为 1Y/3Y/5Y/7Y **名义国债收益率腹部组合**：1Y 看近端政策路径，3Y 看中段再定价，5Y/7Y 看腹部传导。DGS10 名义收益率和 DFII10 实际收益率必须分开命名，仅作长期折现率背景；10Y-2Y/10Y-3M 只作曲线背景。脚本必须预计算各期限 bp 变化及 5Y-3Y、7Y-5Y 斜率，模型不得自行换算或相减。
- 2026-05-21：美元流动性分析中 RRP 必须区分“边际方向”和“存量缓冲垫”：RRP 上升 = 资金回到联储停车场，短期通常偏不利风险资产；RRP 下降 = 资金从联储出来，短期通常偏利好风险资产。但当 RRP 降到很低后，继续下降不再是持续利好，而代表缓冲垫接近耗尽，未来 TGA 补库/QT 冲击更容易直接打到准备金。前端文案必须避免把“RRP上升=缓冲垫恢复”和“释放流动性下降”混在一起造成歧义。
- 2026-05-21：项目结构已重新确认：正式架构必须是"脚本抓取/计算数据 -> model_input.json + prompt -> 模型输出结构化 analysis.json -> 固定前端模板读取 JSON 并确定性渲染"。不得让模型直接输出 `analysis.html` 或完整网页片段；HTML/CSS/JS 属于产品层，只在产品迭代时修改。脚本每次运行应生成 rule_fallback `analysis.json` 防止前端读取旧分析，正式模型分析可覆盖该 JSON。
- 2026-05-25 / 2026-06-18 / 2026-07-01 更新：用户触发词"更新数据并分析"= 完整流水线，必须依次完成：① 运行 `scripts/run.py` 抓最新数据（脚本会将 `analysis.json` 重置为 `pending_model_analysis`，防止旧分析复用）；② **模型（AI agent）自主读取 `model_input.json` + `prompts/usd_liquidity_prompt.md`，生成结构化分析并覆盖 `output/latest/analysis.json`**（此步骤不得跳过，不得留 `rule_fallback` 或 `pending_model_analysis` 占位符）；③ 启动/打开本地前端预览（**2026-07-01 起不再需要手动同步 data.js**，前端 loadJson 会检测占位符并自动 fetch analysis.json）；④ 每次跑完都要自动提交并 push 最新落表数据、`output/latest` 数据和相关代码/提示词修改到 GitHub。不得以 rule_fallback 代替模型分析，不得跳过本地预览和 GitHub push。
- 2026-05-28：美元流动性流水线工作流已更新：`scripts/run.py` 不再调用 `build_fallback_analysis()`，改为脚本跑完后由**模型自主**读取 `model_input.json` + prompt 并生成 `analysis.json`。`write_outputs()` 不再接收 `analysis` 参数，改由模型在流水线中自主完成分析并覆盖 `analysis.json`。`run.py` 执行完后若 `analysis.json` 仍为 `pending_model_analysis` 占位符，则触发模型分析步骤。
- 2026-05-26：项目代码重构完成。原 `scripts/usd_liquidity_brief.py`（2786 行单文件）已拆分为 9 个模块：`run.py`（主入口）、`utils.py`（公共）、`fetchers.py`（抓取）、`signals.py`（信号）、`dashboard.py`（前端JSON）、`analysis.py`（分析）、`charts.py`（图表）、`db.py`（SQLite）、`jpy_carry.py`（日元carry）。旧文件已移入 `scripts/_archive/`。前端 `app.js` 路径改为 `../../output/latest/`。项目根目录起 HTTP server 后访问 `frontend/usd-liquidity-monitor/`。运行命令：`python3 scripts/run.py "触发语" --json`。
- 2026-05-26：美元流动性项目新增“量级量化”约定：利率是价格信号，分析必须结合市场规模/交易量。已加入 `SOFR_VOLUME`、`TBILL_AUCTION_SIZE`、`TBILL_AUCTION_BTC`，并新增衍生信号 `SOFR_VOLUME_IMPACT`（SOFR相对政策锚偏离 × SOFR交易量，粗略日化资金成本量级）和 `TBILL_AUCTION_ABSORPTION`（T-bill拍卖规模 × 认购倍数）。Prompt 必须要求模型不要只看利率/bid-to-cover，而要做“价格 × 规模”判断；缺少这些量级数据时标为数据风险。
