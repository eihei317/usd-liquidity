## 前端"待模型分析"锁死 bug 修复

### 问题
- 前端频繁显示"待模型分析"，即使模型已生成真正的 analysis.json
- 根因：`app.js` 的 `loadJson` 函数只要 `window.ANALYSIS_DATA` 存在（truthy）就直接返回，不 fetch `/output/latest/analysis.json`
- `run.py` 生成 `data.js` 时写入的占位符是 truthy 对象，导致前端"锁死"在占位符上

### 修复（方向A）
- 修改 `frontend/usd-liquidity-monitor/app.js` 的 `loadJson` 函数
- 检查 `window.ANALYSIS_DATA` 是否为占位符（`meta.status === "pending_model_analysis"`），如果是则继续 fetch
- 效果：模型只需写 `analysis.json`，不再需要手动同步 `data.js`
- GitHub 已推送（commit f5fb83e）

### 架构影响
- **旧流程**：run.py 写占位符到 data.js → 模型写 analysis.json → 模型手动同步 data.js → 前端读 data.js
- **新流程**：run.py 写占位符到 data.js → 模型写 analysis.json → 前端检测占位符后 fetch analysis.json
- 消除了"模型手动同步 data.js"这一脆弱步骤
