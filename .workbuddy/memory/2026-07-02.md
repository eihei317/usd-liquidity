## 修复 db schema UNIQUE 漂移导致流水线崩溃

- 现象：`scripts/run.py` 抓取数据后在图表阶段崩溃，`engineered_series_ts` 插入报 `ON CONFLICT clause does not match any PRIMARY KEY or UNIQUE constraint`；`metrics_ts`/`derived_signals_ts` 全部 ingest 跳过（0 inserted）。
- 根因：`db.py` 的 `CREATE TABLE IF NOT EXISTS` 不会升级已存在的旧表。磁盘上 `engineered_series_ts` 等表保留了旧的 `UNIQUE`（如 `UNIQUE(series_id, as_of, snapshot_file)` 三列），而当前 upsert 用两列 `ON CONFLICT(series_id, as_of)`；`dashboard.py` 自己又用三列 `UNIQUE` 重建同一张表，二者不一致。
- 修复：
  - `db.py` 新增 `TABLE_DEFS` + `_ensure_table`/`_heal_schema`：`init_db` 每次启动自检每张表的唯一约束，若与 `TABLE_DEFS` 不符则在保留历史行（按唯一键去重、保留最新 ingested 行）的前提下重建表。一次性修好，且以后 schema 漂移不会再崩。
  - `dashboard.py` 的 `engineered_series_ts` 改回 `UNIQUE(series_id, as_of)`，与 db.py 一致。
- 修复后 `run.py` 跑通：33 metrics + 22 derived_signals 正常 ingest，图表生成成功。
